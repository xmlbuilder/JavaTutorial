# 🧠 컴퓨터와 문자 인코딩 요약
## 1️⃣ 숫자와 문자의 저장 방식 차이
- 숫자: 10진수를 2진수로 변환해 메모리에 저장
- 문자: 수학 공식으로 변환 불가 → 문자 집합(Charset)을 통해 숫자와 매핑

## 2️⃣ 문자 인코딩과 디코딩
- 인코딩: 문자를 숫자로 변환 (예: 'A' → 65)
- 디코딩: 숫자를 문자로 변환 (예: 65 → 'A')

## 🔤 주요 문자 집합

### 🔹 ASCII (1960년대)
- 7비트 = 128문자 (0~127)
- 제어 문자(0126)
- 영어, 숫자, 특수문자만 표현 가능
- 모든 문자 집합의 기반

### 🔹 ISO 8859-1 (LATIN1)
- 8비트 = 256문자
- ASCII + 서유럽 문자 추가 (예: À, Ä, Ç 등)
- ASCII와 호환

### 🔹 EUC-KR (1980년대)
- 한글 2,350자 + 한자 4,888자 + ASCII
- 2바이트 인코딩
- ASCII와 호환

### 🔹 MS949 (1990년대, Windows)
- EUC-KR 확장판
- 한글 11,172자 모두 표현 가능
- ASCII와 호환

## 🌍 유니코드와 UTF 시리즈
### 🔹 유니코드 (1990년대)
- 전 세계 모든 문자를 하나의 문자 집합으로 통합
- 다양한 인코딩 방식 존재 (UTF-8, UTF-16 등)

### 🔹 UTF-16
- 2바이트 고정 + 일부 문자 4바이트
- 대부분의 문자 표현 가능
- 단점: ASCII도 2바이트 사용 → 비효율

### 🔹 UTF-8 (현대 표준)
- 가변 길이 인코딩 (1~4바이트)
- 1바이트: ASCII
- 2~3바이트: 한글, 한자, 일본어
- 4바이트: 이모지, 고대 문자
- 장점:
- ASCII와 완벽 호환
- 저장 공간 절약 (영문 위주 문서에 유리)
- 웹 표준 (W3C, 대부분의 웹사이트 사용)

## 💻 자바에서 문자 집합 다루기
### 사용 가능한 문자 집합 조회
```java
Charset.availableCharsets()
```
- 자바 + OS가 제공하는 모든 문자 집합 확인 가능
### 문자 집합 조회 및 별칭 확인
```java
Charset charset = Charset.forName("MS949");
Set<String> aliases = charset.aliases();
```
- 대소문자 구분 없음
- 예: MS949 = ms949, windows-949, x-windows-949 등
### 자주 쓰는 상수
```java
Charset utf8 = StandardCharsets.UTF_8;
Charset defaultCharset = Charset.defaultCharset();
```


## ✅ 결론
- UTF-8은 현재 가장 널리 쓰이는 문자 인코딩 방식
- ASCII와 호환, 저장 공간 효율, 웹 친화적
- 한글 윈도우는 여전히 MS949를 기본으로 사용하지만, UTF-8로 전환 중
